# 双层记忆系统与记忆流动管理

## 定义

双层记忆系统是一种将 Agent 记忆划分为**短期记忆**和**长期记忆**的架构模式。短期记忆负责捕获当前会话的上下文信息，长期记忆负责持久化存储跨会话累积的核心知识。

记忆流动管理是指在这两层记忆之间建立高效的**信息流转机制**，确保有价值的短期信息能够沉淀为长期记忆，同时保持长期记忆的准确性和时效性。

## 核心机制

### 短期记忆（会话消息）

- 来源：用户与 Agent 的实时交互
- 特点：高频、大量、细节丰富
- 存储：内存或当前会话状态

### 长期记忆（知识文件）

nanobot 的长期记忆由以下文件组成：
- **MEMORY.md** — 系统级记忆，存储核心指令、配置和系统信息
- **SOUL.md** — 灵魂文件，存储核心身份、价值观和长期目标
- **USER.md** — 用户档案，存储用户相关信息和偏好
- **history.jsonl** — 追加式历史总结，由 [[consolidator]] 生成

### 记忆流动流程

```
1. 会话消息产生 → 用户与 nanobot 交互
2. Consolidator 处理 → 轻量级压缩，生成历史总结追加到 history.jsonl
3. Dream 分析 → 定期或手动触发，分析 history.jsonl 与长期记忆差异
4. Dream 编辑 → 手术式编辑长期记忆文件
5. GitStore 版本控制 → 自动提交变更，记录记忆演化历史
```

## 关键技术

### 两阶段处理（Dream）

Dream 采用两阶段处理实现记忆整合：

**阶段一：分析差异**
- 读取 history.jsonl 和长期记忆文件
- 分析历史信息与现有记忆的差异
- 识别需要更新的记忆内容
- 生成结构化分析报告

**阶段二：手术式编辑**
- 基于分析报告生成编辑指令
- 执行精确的文件编辑
- 保持记忆连贯性和结构完整性

### 版本控制（GitStore）

自动提交记忆变更到 Git 仓库，记录记忆演化历史，支持回溯和审计。

### 配置控制

```yaml
memory:
  dream:
    enabled: true
    interval: 3600     # 自动触发间隔（秒）
    max_history_lines: 1000  # 分析的历史行数
```

## 设计原则

1. **分层管理**：短期与长期记忆分离，各司其职
2. **轻量流转**：Consolidator 只做压缩，不做深度分析
3. **精准更新**：Dream 采用手术式编辑，避免破坏性修改
4. **版本可追溯**：GitStore 记录每次记忆变更
